# 🌤️ 毎晩じどうで天気データを集める GitHub Actions テンプレート

**プログラムを常時つけっぱなしにしなくても、GitHub が毎晩かってにデータを集めてCSVに保存してくれる**——そんな仕組みを、コピペだけで作れるテンプレートです。

- ✅ **APIキー登録なし**でそのまま動く（Open-Meteo という無料APIを使用）
- ✅ **サーバー不要・お金かからない**（GitHub Actions の無料枠内）
- ✅ **商用利用OK・規約クリーン**なデータだけを使用
- ✅ Python の標準機能だけ。追加ライブラリなし
- ✅ 天気以外（為替・株価・自分の好きなサイト）にも**差し替えて使える**

> このテンプレの本当の価値は「毎晩じどう実行 → データ収集 → CSV保存 → 自動コミット」という**仕組みの型**です。天気はサンプルで、中身を差し替えれば何にでも応用できます。

---

## 📦 何ができるの？

毎晩きまった時間になると、GitHub が自動で：

1. 指定した地点の天気予報を取得し
2. 気温（最高・最低）と降水量をとりだして
3. `data/weather.csv` に1行ずつ追記して
4. リポジトリに自動でコミット（保存）します

あなたは何もしなくて大丈夫。毎日データがたまっていきます。

CSV はこんな感じで増えていきます：

```
date,place,temp_max,temp_min,precipitation,collected_at
2026-07-11,Tokyo,31.2,24.8,0.0,2026-07-10T21:39:40+00:00
2026-07-12,Tokyo,30.8,25.1,5.2,2026-07-11T21:39:41+00:00
```

## 🚀 使い方（小学生でもできるレベルで説明します）

### ステップ1：このリポジトリを自分のものにコピーする

1. このリポジトリの右上の **「Use this template」**（またはフォーク）ボタンを押す
2. 自分のアカウントに新しいリポジトリができます

> 💡 テンプレートじゃなくZIPで受け取った場合は、GitHubで新しいリポジトリを作って、中身のファイルをぜんぶアップロードすればOKです。

### ステップ2：Actions を有効にする

1. コピーしたリポジトリの上部メニューから **「Actions」** タブを開く
2. 「I understand my workflows, go ahead and enable them」的なボタンが出たら押す

これで自動実行の準備完了です。

### ステップ3：まず手動でテスト実行してみる

いきなり夜まで待つ必要はありません。ちゃんと動くか今すぐ試せます。

1. **「Actions」** タブを開く
2. 左のメニューから **「nightly-weather-collect」** を選ぶ
3. 右のほうにある **「Run workflow」** ボタンを押す → もう一度 **「Run workflow」**
4. 数十秒待つと実行がおわり、`data/weather.csv` ができています🎉

### ステップ4：あとは放置でOK

以降は**毎晩じどうで**データが増えていきます。何もする必要はありません。

## ⚙️ カスタマイズ（自分ようにかえる）

変えるファイルは基本 **`src/collect.py` の一番上の `CONFIG` だけ**です。

### 📍 地点をかえたい

`src/collect.py` の CONFIG の緯度・経度をかえるだけ：

```python
CONFIG = {
    "latitude": 34.6937,     # ← ここを大阪の緯度に
    "longitude": 135.5023,   # ← ここを大阪の経度に
    "place_name": "Osaka",   # ← 名前もかえる
    ...
}
```

緯度・経度は「（都市名） 緯度 経度」で検索すればすぐ出ます。

### 🕒 実行する時間をかえたい

`.github/workflows/collect.yml` の `cron` をかえます。

```yaml
schedule:
  - cron: "0 14 * * *"   # UTC 14:00 = 日本時間23:00
```

⚠️ **重要**：cron の時間は **UTC（世界標準時）** です。
日本時間にしたい時刻から **9時間ひいた** 数字を書きます。

| 日本時間で動かしたい | cron に書く数字 |
|---|---|
| 毎晩 23:00 | `0 14 * * *` |
| 毎朝 7:00 | `0 22 * * *` |
| 毎日 正午12:00 | `0 3 * * *` |

### 📊 取る項目をふやしたい

CONFIG の `daily` にカンマ区切りで足します（湿度・風速など）：

```python
"daily": "temperature_2m_max,temperature_2m_min,precipitation_sum,windspeed_10m_max",
```

使える項目一覧 → [Open-Meteo 公式ドキュメント](https://open-meteo.com/en/docs)

## 🔧 応用：天気いがいのデータを集めたいとき

このテンプレの「仕組み」はそのまま使えます。変えるのは `src/collect.py` の中の
**「データを取ってくる部分」と「取りだす部分」だけ**です。

- 為替レート → 無料の為替APIに差し替え
- 自分のブログのアクセス数 → その集計元に差し替え
- 公開されている統計データ → 取得先URLを差し替え

「取得 → 整形 → CSVに追記 → 自動コミット」の流れは共通なので、
`fetch_weather()` と `extract_rows()` の2つの関数を書きかえるだけで応用できます。

## ❓ よくあるハマりどころ

**Q. 自動コミットで権限エラーが出る**
→ リポジトリの Settings → Actions → General →「Workflow permissions」で
**「Read and write permissions」** をONにしてください。

**Q. 決めた時間ぴったりに動かない**
→ GitHub の cron は数分〜十数分ずれることがあります（GitHub 側の仕様）。
きっちり定時である必要がなければ気にしなくてOKです。

**Q. しばらく放置してたら自動実行が止まった**
→ GitHub の仕様で、60日間リポジトリに動きがないと定期実行が停止します。
たまにコミットするか、Actions画面から手動実行すれば復活します。

**Q. お金はかかる？**
→ パブリックリポジトリなら Actions は無料。プライベートでも
無料枠（月2,000分）内でじゅうぶんおさまります。
